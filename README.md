# JoinRender-Flow

做这个工具的原因很简单：ComfyUI 太难用了。

不是说它不好，功能确实强大，但每次打开看到那一堆节点就头疼。什么 Latent、VAE、CLIP，光搞懂这些概念就得花好几天。我们就想，能不能把这些东西包一层，让普通人也能用？

所以就有了这个项目。

## 大概思路

把节点分成三类就够了：

- **输入** - 文字、图片、视频，你的素材从这进
- **LLM** - 相当于大脑，帮你把"我想要一个赛博朋克风格的角色"这种话变成模型能理解的提示词
- **媒体生成** - 真正干活的，出图出视频

角色一致性是个老问题了。我们的做法是搞一个"角色参考"节点，先生成一张标准的 T-Pose 图，后面所有生成节点都可以引用这张图。节点上有个 🔒 标记的输入口就是干这个的。

## 跟 ComfyUI 比

说实话比不了，人家那个灵活性我们肯定没有。但我们的目标用户不一样：

- ComfyUI 适合技术向的人，想怎么折腾怎么折腾
- 我们这个适合设计师、创作者，能快速出活就行

另外我们是纯前端，不用装 Python 环境，打开浏览器就能用。

## 能干啥

**接 API：**
- OpenAI（LLM、图像分析）
- Replicate（SDXL 出图、SVD 出视频）
- 本地跑 Ollama 也行
- 没 API 也能用，有个模拟模式测试工作流

**工作流：**
- 存成 JSON，下次接着用
- 能导入 ComfyUI 的工作流，自动转格式
- 也能导出成 ComfyUI 格式

**模板：**
内置了几个常用的，不用从零开始搭：
- 文生图
- 图生视频
- 风格迁移
- 角色一致性流水线
- 还有几个，自己看吧

## 节点

输入类：文本、图片上传、视频上传

LLM 类：LLM 处理、提示词增强、图像分析、JSON 分离

生成类：角色参考、图像生成、视频生成、首尾帧插值、图像变体、风格迁移、背景移除、图像放大

输出类：图像输出、视频输出、分镜板

## 跑起来

```bash
npm install
npm run dev
```

然后浏览器开 http://localhost:5173

操作很简单：
1. 左边拖节点到画布
2. 连线（输出口在右边，输入口在左边）
3. 点播放按钮

要用真实 API 的话，点左下角「设置」填 Key。

## 技术栈

React + TypeScript + Vite + Zustand

没有后端，纯前端项目。

